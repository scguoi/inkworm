# Courses YYYY-MM Subdirectory Layout — Design Spec

> **Status**: Design
> **Date**: 2026-05-06
> **Parent spec**: `2026-04-21-inkworm-design.md` (storage §)

## 0. Scope

Reorganize the on-disk courses library from a flat layout to a one-level
month-bucketed layout, plus a one-shot startup migration:

```
courses/
  2026-05/
    06-context-management-in-claude-code.json
  _legacy/                  # only created if a malformed-id file is found
```

Goal: keep `courses/` browsable as the library grows. `course.id` is **not
changed**; the path is mechanically derived from the id, so all existing
references in `mistakes.json` / `progress.json` / etc. keep working with
zero data migration.

Out of scope: file format changes (stays JSON), id format changes,
multi-level nesting beyond `yyyy-mm`, and a manual `migrate` subcommand.

---

## 1. On-Disk Layout

### 1.1 Path derivation

`course.id` is generated by `llm::reflexion::unique_id` as
`yyyy-mm-dd-<slug>` (e.g. `2026-05-06-context-management-in-claude-code`).
The on-disk path is derived purely from the id:

| Component | Source | Example |
|---|---|---|
| Subdirectory | `id[0..7]` | `2026-05` |
| File name | `id[8..]` + `.json` | `06-context-management-in-claude-code.json` |
| Full path | `<courses_dir>/<yyyy-mm>/<dd-rest>.json` | `courses/2026-05/06-context-management-in-claude-code.json` |

`dd-rest` always begins with two digits because `unique_id` formats the
date with `chrono`'s `%Y-%m-%d` (zero-padded by construction).

### 1.2 New helper

Add to `src/storage/course.rs` (private):

```rust
fn course_path(courses_dir: &Path, id: &str) -> Result<PathBuf, StorageError> {
    // Length + prefix shape guard (defense-in-depth; validate() also enforces).
    if !has_yyyy_mm_dd_prefix(id) {
        return Err(StorageError::InvalidId(id.to_string()));
    }
    let yyyy_mm = &id[0..7];                 // "2026-05"
    let file = format!("{}.json", &id[8..]); // "06-context-management-in-claude-code.json"
    Ok(courses_dir.join(yyyy_mm).join(file))
}
```

`StorageError` gains a new variant:

```rust
#[error("invalid course id (must match yyyy-mm-dd-<slug>): {0:?}")]
InvalidId(String),
```

`load_course`, `save_course`, `delete_course` switch from
`courses_dir.join(format!("{id}.json"))` to `course_path(courses_dir, id)?`.
The function is fallible because callers may pass an unvalidated id (e.g.
a stale id from an external caller); the regex guard makes the byte-slice
indices safe and surfaces the error instead of panicking.

`save_course` additionally calls `std::fs::create_dir_all(parent)` before
the atomic write so the `yyyy-mm/` directory is created on first use.

### 1.3 `list_courses` scan rules

Scan exactly one level: enumerate immediate subdirectories of
`courses_dir`, skip `_legacy/` and any non-directory or non-`yyyy-mm`-shaped
entry, then enumerate `*.json` inside each.

The `id` for each discovered file is reconstructed by joining the parent
directory name and the file stem with a hyphen:

```text
parent_dir = "2026-05"
file_stem  = "06-context-management-in-claude-code"
id         = "2026-05" + "-" + "06-context-management-in-claude-code"
           = "2026-05-06-context-management-in-claude-code"
```

This avoids re-parsing every JSON file just to enumerate. The
reconstructed id is then passed through `Course::validate()` shape checks
when the file is actually loaded; mismatches between path-derived id and
the in-file `course.id` field are treated as malformed and skipped (with
a `tracing::warn!` log, not a `boot_warning` — listing must not fail).

Acceptance criteria:
- Files in `courses_dir` root are ignored (after migration the root holds
  no `*.json`; if the user manually drops one in, it's invisible until
  moved).
- Files inside `_legacy/` are ignored.
- Files inside an arbitrarily-named subdirectory (e.g. `tmp/`) are ignored.
- Subdirectory names that don't match `^\d{4}-\d{2}$` are ignored
  (defensive — `migrate` shouldn't produce these, but a stray dir
  shouldn't crash listing).

## 2. Validation Tightening

### 2.1 New validation rule

Extend `Course::validate()` (`src/storage/course.rs`):

```rust
if !has_yyyy_mm_dd_prefix(&self.id) {
    errs.push(ValidationError::IdMissingDatePrefix(self.id.clone()));
}
```

where `has_yyyy_mm_dd_prefix` is a `pub(crate)` byte-level helper that
checks `^\d{4}-\d{2}-\d{2}-` without taking a `regex` crate dependency
(ASCII digits only; the id is always ASCII kebab-case by construction).
The same helper is reused by `course_path` and `src/storage/migrate.rs`.

### 2.2 New `ValidationError` variant

```rust
#[error("id is missing yyyy-mm-dd- prefix: {0:?}")]
IdMissingDatePrefix(String),
```

The existing kebab-case check stays; the new check is additive.

### 2.3 Why early-fail

Path derivation depends on the `yyyy-mm-dd-` prefix. Allowing a
non-conforming id to reach `course_path` would compute a wrong path and
silently misplace the file. Validating up front in the same place we
already check kebab-case keeps storage invariants tight.

## 3. Startup Migration

### 3.1 When

`src/main.rs` runs `migrate_courses_to_yyyy_mm` once during boot, **before**
`list_courses` / `prune_orphans` so the rest of startup observes the new
layout.

### 3.2 Algorithm

```text
for each direct child of courses_dir:
    if it's a directory: skip
    if it's not *.json: skip
    parse the file as a Course
    if parse fails or id does NOT match ^\d{4}-\d{2}-\d{2}-:
        rename file to courses_dir/_legacy/<original-name>
        push a boot_warning
        continue
    target = course_path(courses_dir, id)
    if target already exists:
        rename source file to _legacy/<original-name>
        push a boot_warning ("conflict")
        continue
    create_dir_all(target.parent())
    rename source -> target
```

Renames are `std::fs::rename` (atomic within the same filesystem; the
config dir is on one filesystem in practice).

### 3.3 Boot warnings

Aggregate into `boot_warnings` (already used elsewhere in `main.rs`).
Two distinct warning shapes:

- `"Moved {n} malformed course file(s) to _legacy/ — please review"`
- `"Moved {n} course file(s) to _legacy/ due to path conflicts — please review"`

A successful, conflict-free migration is silent (no warning). Only the
counts that needed user attention surface to the UI.

### 3.4 Failure handling

Any IO error during migration (read, mkdir, rename) is **fatal**: the
function returns the error and `main` exits with a non-zero status,
printing the error to stderr. Rationale: continuing after a partial
migration risks half-moved state that's harder to reason about than a
clean abort. The user fixes the disk problem, restarts, and migration
reruns idempotently.

### 3.5 Idempotence

After a successful run, the courses root has no top-level `*.json`. The
loop in §3.2 finds nothing to do and returns immediately. Repeated
restarts are safe and cheap (one `read_dir` and zero work).

## 4. Affected Modules

| Module | Change |
|---|---|
| `src/storage/course.rs` | Private `course_path()`; `load_course` / `save_course` / `delete_course` use it; `save_course` does `create_dir_all(parent)`; `list_courses` scans `<dir>/<yyyy-mm>/*.json`; `validate()` adds prefix check; `ValidationError::IdMissingDatePrefix` variant |
| `src/main.rs` | New `migrate_courses_to_yyyy_mm()` invoked early in boot; aggregates into `boot_warnings`; fatal on IO error |
| `src/storage/mistakes.rs` | **Unchanged** (id unchanged) |
| `src/storage/progress.rs` | **Unchanged** |
| `src/llm/reflexion.rs` | **Unchanged** (`unique_id` already produces the required prefix) |
| Existing course-storage tests | Update fixture paths and assertions to the new layout |

## 5. Testing

### 5.1 New unit tests

- `migrate: well-formed flat file moved to yyyy-mm/`
- `migrate: file with malformed id moved to _legacy/ + boot_warning`
- `migrate: file that fails to parse moved to _legacy/ + boot_warning`
- `migrate: same-name conflict (target exists) moves source to _legacy/`
- `migrate: noop when courses_dir is already laid out`
- `migrate: idempotent across two consecutive runs`
- `course_path: derives expected yyyy-mm/dd-rest.json`
- `Course::validate rejects id without yyyy-mm-dd- prefix`
- `list_courses: skips _legacy/, root-level json, and non-yyyy-mm dirs`

### 5.2 Regression coverage

`load_course → save_course → load_course` round-trip with a freshly
generated id, asserting the file lives at `<dir>/yyyy-mm/dd-rest.json`.

## 6. Migration Result for Current User

Given the current state at `~/.config/inkworm/courses/`:

```
2026-05-06-context-management-in-claude-code.json   (well-formed)
```

After first boot post-upgrade:

```
2026-05/
  06-context-management-in-claude-code.json
```

No `_legacy/` directory created. No `boot_warnings`. The mistakes book
keeps `2026-05-06-context-management-in-claude-code|1|2 → 1` working
because the id didn't change.
